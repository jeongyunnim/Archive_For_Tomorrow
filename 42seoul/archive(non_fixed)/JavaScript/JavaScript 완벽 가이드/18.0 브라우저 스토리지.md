# 브라우저  스토리지란?
### 서버 스토리지
- 서버측 데이터베이스
- 필수적, 영속적 데이터를 저장
## 브라우저 스토리지
- 종류
	- 로컬 스토리지, 세션 스토리지
	- 쿠키
	- IndexedDB
- 용도
	- 임시적이고 편의성을 위한 데이터를 저장
	- 인증 데이터
### 로컬 및 세션 스토리지
- 키-값을 저장
	- 복잡한 데이터는 저장하기 힘들다는 단점이 있다.
- 파일에 저장 된다.
- **세션 스토리지**
	- 분석 서버로 전송해야 하는 분석 데이터
	- 사용자의 세션 ID
- **로컬 스토리지**
	- 사용자 설정
	- 기본 사용자 데이터 관리
	- 브라우저에서 작동하는 JS코드만 로컬 스토리지와 통신이 가능하다.
### 쿠키
- 키-값을 저장
	- 만료 일자를 지정하여 자동으로 삭제되도록 처리할 수 있다.
	- 로컬 스토리지는 자바스크립트로 로직을 추가해야 한다.
- 엔트리 중 하나에 옵션으로 설정할 수 있다.
- 용도 
	- 기본 설정
	- 세션 ID 등을 관리하는 데 사용한다.
- 자바스크립트에서 접근하거나 지울 삭제할 수 있다.
- 작업하기에 좋지 않으나, 활용도가 높다.
	- 만료 날짜
	- HTTP 요청과 함께 서버로 전송된다.
### IndexedDB
- 클라이언트 측 데이터베이스
- 브라우저에 빌트인 되어있다.
	- 쿼리 언어를 사용할 수 있다.
	- 복잡한 데이터를 관리할 수 있다.
- 용도
	- 복잡하지만 중요하지 않은 데이터를 저장
- 자바스크립트로 관리할 수 있다.
### 주의
- 사용자가 임의로 지울 수 있기 때문에 브라우저 스토리지에 의존적으로 작성해서는 안 된다.
